# Metric Databases: Towards Embedding-independent Vector Search

## Principal Goal

Develop a prototype vector database system that is agnostic to the embedding models.

This project has been proposed by Yang Cao (`Yang.Cao@ed.ac.uk`).

## Background

Vector databases such as Milvus, pgvector, and Pinecone currently exist as isolated vector data silos because each vector database is effectively tied to a specific embedding model. A vector generated by one model, such as an OpenAI model, cannot be directly compared to a vector generated by another model, such as a Mistral model. This prevents data integration across vector stores.

The goal of this project is to build **MetricDB**, a database system that decouples **logical similarity** from **physical embedding**. Based on recent and ongoing research, different embedding spaces can be locally isometric, which makes it possible to align, transform, and query across disparate vector stores. This allows vector databases to be lifted into metric databases, which can then be mapped to a federation of vector databases produced by different embedding models.

## Overall Objective

Build a functioning prototype of MetricDB.

The project is divided into five independent but complementary sub-projects. Students will share a common codebase but will be assessed solely on their own module. This is part of an ongoing research project involving one staff member and three PhD students. Students are welcome to collaborate with them.

## Sub-projects

### 1. Predicate Vector Search

#### Objective

1. Survey and benchmark methods for predicate vector search, that is, search over hybrid relation-vector data with both relational predicates and nearest-neighbor search over vectors.
2. Optional: develop a predicate vector search method specifically for metric databases, viewed as a federation of multiple vector databases using different embedding models.

### 2. Metric (SQL) Query Parser and Logical Optimiser

#### Objective

1. Extend an SQL parser to support metric search operations over metric databases.
2. Optional: develop query rewriting rules and logical optimization for such queries.

### 3. Virtual and Hard Unions of Vector Databases

#### Objective

Implement and benchmark two methods for querying a metric database, in the simple case defined as the union of two different vector databases:

- **Soft union**: query the databases independently by transforming queries.
- **Hard union**: integrate the databases into one vector space and then query them jointly.

These two methods have already been developed conceptually in prior research publications. This thesis will make them practical and robust in MetricDB.

### 4. From Vector Indexing to Metric Indexing

#### Objective

Extend vector indices to metric indices.

Two possible approaches:

1. Build the metric index from scratch after integrating embedding vectors from different databases.
2. Transform indices and merge them from one database into another.

These are ongoing research topics in the research group. Some algorithms and techniques already exist, but they still need to be adapted and implemented in MetricDB so they work with other system components.

### 5. Geometric Hashing of Embedding Vectors for Vector-based Entity Linking

#### Objective

MetricDB relies on reference mappings, also called anchor points, between vector databases in order to link and integrate them into a metric view. This sub-project focuses on discovering such reference mappings between two vector databases.

The goal is to develop techniques to link vectors across vector databases, that is, identify vectors encoding the same objects, and then implement and benchmark these techniques in MetricDB. Ongoing research in the group will support the conceptual algorithmic design, while this sub-project will focus on making the methods work well inside MetricDB.
