# Proposal 第 1-5 部分修改方案

## 目标

当前阶段在没有代码的前提下，优先精修 proposal 的前半部分，即 `1. 研究背景与动机` 到 `5. 相关工作`。这一轮的目标不是继续扩展后半部分，而是让前半部分在问题定义、文献支撑和 related work 组织上更稳、更具体、更适合拿去和老师讨论。

## 需要修改的核心问题

### 1. `4.3 可行性` 的文献支撑来源需要区分清楚

当前写法容易让读者误解为“主线和扩展线都已有同等强度的文献支撑”。这一点需要改成分层表达：

- 上游前提的支撑，主要来自老师方向核心论文；
- 主线 `重建 metric index` 的支撑，主要来自 HNSW、FAISS 以及更直接的 metric indexing 文献；
- 扩展 `已有 index 的转换与合并` 只能写成有相关研究启发，但仍然是更开放的研究问题。

### 2. `5.2` 中“MetricDB 的研究背景指出”表述过虚

这一句不应继续写成笼统背景判断，而应该直接点名具体论文。当前最合适的直接依据是：

- `Integrating Vector Databases across Embedding Models`

必要时也可配合 `Word Translation Without Parallel Data` 和 `The Platonic Representation Hypothesis`，但句子的主语应以具体论文作者或论文本身为主，而不是“研究背景”。

### 3. `5.1 - 5.3` 中的代表文献还需要更明确说明“各自讲了什么”

当前相关工作已经有结构，但有些地方还是偏“列名字”。下一轮需要让每类代表文献都承担明确功能：

- `HNSW / FAISS`：说明传统单空间 vector indexing 的成功路线；
- `Yang / Cao / Ren`：说明异构向量数据库整合与统一 metric view 的上游前提；
- `LIMS / pivot selection survey`：说明 metric-space indexing 有独立脉络，不是普通 ANN 的简单改写。

### 4. `5.3` 结尾不宜只列文献名

最后一段不应该变成文献目录。更好的写法是：

- 在 `5.1 / 5.2 / 5.3` 对应位置各自解释代表文献的作用；
- `5.3` 最后一段只保留总结性概括，不再长串列出文献名。

### 5. 当前修改范围只限定在 `1-5`

这一轮不继续深改 `6-10`。它们保留现状，不删、不重写，只在前半部分逻辑和文献支撑收稳后再处理。

## 具体实施措施

### A. 修改 `4.3 可行性`

- 把“两个方向都有文献支撑”改成分层写法；
- 明确：
  - 老师方向核心论文支撑上游整合前提；
  - HNSW / FAISS / LIMS 等支撑主线可行性；
  - 扩展路线只有相关启发，不写成成熟已有方案。

### B. 修改 `5.2`

- 删除或改写“MetricDB 的研究背景指出”这种笼统表达；
- 改成以具体论文为主语；
- 强化 `Integrating Vector Databases across Embedding Models` 在这一节中的中心位置。

### C. 修改 `5.1`

- 在现有 HNSW / FAISS 基础上，加一句说明它们代表“单一 embedding 空间”下的传统成功路线；
- 接着补一小段，引入 `LIMS` 和 `pivot selection survey`，说明 metric-space indexing 本身有独立研究脉络。

### D. 修改 `5.3`

- 保留三点迁移挑战；
- 在结尾把“文献列表式总结”改成“问题导向式总结”；
- 用一句话概括：
  - 哪类文献支撑传统 vector indexing；
  - 哪类文献支撑 metric-space indexing；
  - 为什么这两类工作共同导向本 proposal。

### E. 同步更新结构复核清单

- 把 `1-5` 中已完成的修正同步登记到 `Proposal结构复核清单.md`；
- 将 `文献与引用` 这一类的状态从“开始补直接文献”推进到“related work 已开始引入 metric-space indexing 脉络”。

## 执行顺序

1. 先修改 `4.3`；
2. 再修改 `5.1`；
3. 再修改 `5.2`；
4. 再修改 `5.3`；
5. 最后更新 `Proposal结构复核清单.md`。
