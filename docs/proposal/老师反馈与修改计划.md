# 老师反馈与修改计划

保存日期：2026-04-19

## 老师反馈要点

### 1. 引用问题

- reference section 中列出的论文必须在正文中显式引用。
- 每一篇列入 bibliography 的论文，都应在 related research、background、motivation 或首次提到对应工作时被 citation 引用。
- 如果使用 LaTeX，这一点尤其需要避免，否则 reference list 与正文脱节会显得不正常。

### 2. Sec 2.1 核心问题不够清楚

老师认为原表述：

> Given multiple heterogeneous vector databases that have already been aligned or integrated into a unified metric view, how should we design and implement an efficient metric index to support nearest-neighbour search and related query operations?

读起来不够清楚。

需要澄清：

- 项目的 primary goal 是否只是对 aligned / transformed / unioned vector dataset 构建一个 vector index，例如 HNSW；
- 如果只是调用现有 HNSW / FAISS 库在 unioned vector set 上建索引，那么工作量太小，不足以支撑 thesis；
- 需要说明相比简单调用现有 vector index library，额外的技术工作和研究贡献是什么。

### 3. 真正更有技术性和新颖性的部分

老师指出更有技术/研究价值的方向应是：

- 对原始 embedding vectors 上已经构建好的 HNSW / IVF 等索引进行 transform；
- 将多个原始子集上的 index 转换、拼接或合并；
- 最终形成 transformed union vector set 上的有效 index；
- 将这种方法与简单地在 unioned integrated vector set 上 rebuild index 的 baseline 进行比较。

因此，原 proposal 中把 "exploring index conversion and possible merging only if time permits" 写成 optional 会削弱 thesis 的研究贡献。

### 4. Sec 2.2 中 metric index 与 vector index 的区别不清楚

- 需要澄清当前语境中的 "metric index" 与常规 vector index，例如 IVF、HNSW，究竟有什么区别。
- 如果讨论的是 HNSW / IVF 这类方法，最好直接称为 "vector index"，避免术语混乱。

### 5. Sec 2.3 和 Sec 3 需要重新定位主线

- 如果 index conversion and merging 只是 optional，那么 proposal 会让人觉得主线只是调用现有库建索引。
- 这不足以支撑 thesis。
- 因此应把 index conversion and merging 改成 thesis project 的 main focus。
- 简单地在 aligned and unioned vector set 上重建 vector index，应作为 baseline method。

## 需要修改 proposal 的方向

1. 把项目主线从 "build a relation-level metric index prototype" 调整为 "transforming, converting, and merging pre-built vector indices across aligned embedding spaces"。
2. 将 "rebuilding an index over the aligned and unioned vector set" 明确写成 baseline。
3. 删除或弱化 "conversion and merging only if time permits" 的表述，避免把真正的研究贡献写成 optional。
4. 在 motivation 后增加 background / corresponding work 小节，集中介绍 HNSW、IVF、FAISS、vector index rebuilding、cross-space alignment 等基础。
5. 将 "metric index" 术语改清楚：
   - 如果指 HNSW / IVF / FAISS 这类方法，使用 "vector index"；
   - 如果指更抽象的 distance-based metric-space indexing，再单独定义。
6. 检查 LaTeX 正文中的所有 bibliography entries，确保每篇 reference 都有对应 `\cite{...}`。

- 承认原 proposal 将项目写得过于像 systems engineering。
- 明确接受老师建议：index conversion and merging 会成为 main focus。
- baseline 是 rebuild a vector index over the aligned and unioned vector set。
- 会澄清 technical contribution：不是简单调用 HNSW / FAISS，而是研究如何 reuse, transform, and merge pre-built indices from original embedding spaces.
- 会统一术语：HNSW / IVF / FAISS 称为 vector index；只有在抽象 distance-based 语境下才使用 metric index。
- 会修正 citation 问题，确保 reference list 中每篇论文都在正文中被引用。
